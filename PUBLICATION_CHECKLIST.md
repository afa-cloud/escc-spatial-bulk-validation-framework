# Publication checklist

Current connector status:

- Repository metadata files have been created through the GitHub connector.
- The full code release package has been prepared locally as `escc-spatial-bulk-validation-framework_v1.0.0-submission.zip`.
- The local package contains the complete S2 Code scripts, manifests, requirements, citation metadata, Zenodo metadata, and processed supporting tables.

Manual account-level steps still required:

1. Change repository visibility to Public if this repository is to be cited in the manuscript.
2. Upload or push the complete local release package contents to this repository.
3. Create a GitHub release named `v1.0.0-submission`.
4. Connect the repository to Zenodo and publish/archive the GitHub release.
5. Add the Zenodo DOI to the manuscript Code Availability statement.

Do not upload full-text PDFs, temporary Word lock files, private notes, controlled-access data, API keys, or unpublished third-party data.
